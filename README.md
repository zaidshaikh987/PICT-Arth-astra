### 3. AI & Agentic Intelligence

ArthAstra‚Äôs intelligence is implemented in layers, backed by deterministic math:

1. **Perception & Knowledge (Vision + RAG)**  
   - Forensic document vision with `gemini-2.5-flash` to inspect Aadhaar/PAN/bank statements.  
   - RAG over `lib/knowledge/company-brain.ts` (RBI rules, documentation requirements, agentic architecture).

2. **Collaborative Planning ‚Äì Recovery Squad (ADK pipeline)**  
   - **üïµÔ∏è Investigator:** analyzes root causes with tools like `calculateDTI`, credit simulators and CIBIL fetcher.  
   - **üê∫ Negotiator:** constructs negotiation scripts and lender‚Äëfriendly narratives.  
   - **üèóÔ∏è Architect:** builds a stepwise, time‚Äëbound recovery roadmap.

3. **Adversarial Judgment ‚Äì Financial Council (ADK debate)**  
   - **‚ö° Optimist:** argues for approval based on strengths and future potential.  
   - **üîí Pessimist:** argues for rejection based on risk policies.  
   - **‚öñÔ∏è Judge:** weighs both sides and outputs a binding verdict (JSON‚Äëlike structure used by UI).

4. **Global Chat Orchestrator**  
   - `/api/chat` uses an Orchestrator Agent to route each user message to:  
     `ONBOARDING`, `LOAN_OFFICER`, `RECOVERY` or `GENERAL` based on intent.  
   - Responses are short, RAG‚Äëgrounded, and aware of the user profile when available.

### 4. Voice & Accessibility

**Bilingual Voice Assistant 2.0**

- Embedded voice assistant orchestrated via `lib/voice-assistant-context.tsx`.  
- Registration of form fields (id, label, question) enables fully voice‚Äëdriven onboarding.  
- Navigation commands (e.g. "open eligibility", "show documents") map to dashboard routes.

Hybrid flow:

1. **Continuous feedback:** browser Speech API for instant on‚Äëscreen transcription while speaking.  
2. **High‚Äëaccuracy finalization:** `/api/voice-transcribe` sends audio blobs to Gemini 2.5 Flash for accurate text (Hindi/English mix, numeric strings, names).

## üß± Architecture & Data Flow

High‚Äëlevel lifecycle:

1. **Onboarding & Registration**
   - User completes steps 1‚Äì5 in `/onboarding`.  
   - Frontend posts to `/api/user/register`.  
   - Backend:
     - connects to MongoDB (`connectDB()`),  
     - normalizes phone, initializes `creditScoreHistory`, sample `emiSchedule`, `spendingCategories`,  
     - persists `User`, creates welcome alerts, sets `userId` httpOnly cookie.

2. **Login & Session**
   - `/login` posts to `/api/user/login`.  
   - If phone exists, `sessionTs` updated and `userId` cookie is set.  
   - `UserProvider` fetches `/api/user/me` on mount and provides `user`, `updateUser`, `refreshUser`.

3. **Dashboard Rendering**
   - Layout: `DashboardLayout` wraps all `/dashboard/*` pages.  
   - `DashboardOverview` pulls `user` and runs `calculateDetailedEligibility`, `calculateDTI`, `analyzeEmploymentRisk`.  
   - UI cards (eligibility gauge, docs progress, approval odds) are derived purely from stored user + calculator outputs.

4. **AI Calls**
   - Frontend widgets call APIs:
     - `/api/eligibility-insights` ‚Üí specialist loan officer agent over deterministic stats.  
     - `/api/rejection-recovery` ‚Üí ADK Recovery Squad pipeline.  
     - `/api/council-meeting` ‚Üí ADK Financial Council debate.  
     - `/api/document-analyzer` ‚Üí Gemini Vision doc analysis.  
     - `/api/chat` ‚Üí orchestrated chatbot with RAG and tools.

5. **Alerts & Notifications**
   - `Alert` model stores user‚Äëlevel alerts.  
   - `/api/alerts` exposes fetch + mark‚Äëread.  
   - `/api/alerts/generate` and `/api/cron/process-dropoffs` create alerts and optional WhatsApp nudges for drop‚Äëoffs or credit changes.  
   - `/api/notify` formats Twilio WhatsApp/SMS messages for key stages (docs uploaded, credit check started/completed, lender matches).

## üõ†Ô∏è Getting Started (Local)

### 1. Clone & Install

```bash
git clone https://github.com/zaidshaikh987/PICT-Arth-astra.git
cd PICT-Arth-astra/arthastra
npm install
```

### 2. Environment Variables

Create `.env.local` in `arthastra/`:

```env
# Core
GOOGLE_GENERATIVE_AI_API_KEY=your_primary_gemini_key
# Optional: comma‚Äëseparated list for key rotation
GOOGLE_API_KEYS=key1,key2,key3

MONGODB_URI=your_mongodb_connection_string

# Optional: Twilio for WhatsApp/SMS notifications
TWILIO_ACCOUNT_SID=your_twilio_sid
TWILIO_AUTH_TOKEN=your_twilio_token
```

At minimum you need `GOOGLE_GENERATIVE_AI_API_KEY` and `MONGODB_URI`; Twilio is optional.

### 3. Run Dev Server

```bash
npm run dev
```

The app starts on `http://localhost:3000`.  
Use `/onboarding` to create a profile, then `/login` to access the dashboard.

### 4. Helpful Scripts

```bash
# Run type‚Äëaware Next.js build
npm run build

# Lint the project
npm run lint

# Start production server (after build)
npm start

# Test MongoDB connectivity using .env.local
npx ts-node scripts/test-db.ts
```
